# ExpiredIs — Backend API ⚙️

Серверная часть приложения **ExpiredIs** для контроля сроков годности. Проект написан на Node.js и Express, использует SQLite в качестве базы данных и содержит встроенную автоматизацию для ежедневного мониторинга просрочки.

## 🛠️ Технологический стек

* **Runtime:** Node.js
* **Фреймворк:** Express
* **База данных:** SQLite (`sqlite3`)
* **Авторизация:** Passport.js (`passport-jwt`) & `bcrypt`
* **Безопасность & Валидация:** `express-validator`, `express-rate-limit` (защита от брутфорса), `cors`
* **Автоматизация:** `node-cron`
* **Email-сервис:** Resend API

---

## ⏰ Автоматизация и Крон (`node-cron`)

Внутри сервера запущен фоновый планировщик задач, который работает по следующей логике:
* **Интервал:** Запускается автоматически **каждый день**.
* **Логика:** Сканирует базу данных SQLite, находит товары, у которых срок годности истекает ровно **за 3 дня**.
* **Действие:** Формирует список и отправляет email-уведомление пользователю через интеграцию с **Resend API**.

---

## 🌐 API Эндпоинты

### Авторизация (Публичные)
* `POST /register` — Регистрация нового аккаунта (валидация через `express-validator`).
* `POST /login` — Аутентификация пользователя, возвращает JWT-токен при успехе.

### Управление товарами (Защищены Passport JWT)
* `GET /items` — Получить список всех товаров текущего пользователя.
* `POST /items` — Добавить новый товар.
* `DELETE /items` — Удалить товар из системы.

## 🚀 Быстрый запуск

### 1. Установка dependencies
```bash
npm install
```

### 2. Настройка переменных окружения
Создайте файл `.env` в корневой директории бэкенда:
```env
PORT=5000
JWT_SECRET_KEY=your_super_secret_jwt_key
SALTED_ROUNDS=12
ALLOW_ORIGINS=http://localhost:5173,http://localhost:3000
RESEND_API_KEY=re_your_resend_api_key
EMAIL_FROM=ExpiredIs <noreply@yourdomain.com>
```

### 3. Запуск сервера
```bash
npm start
```
Сервер запустится и будет доступен по адресу: `http://localhost:5000`
